# 搭建 07 - 首页自然摆放和交互

这一篇是给 `Home.scene` 微调用的。

你现在不用急着写代码，主要在 Cocos 里拖节点就行。

## 先看节点

打开：

```text
assets/scenes/Home.scene
```

Hierarchy 里重点看这些：

```text
Canvas
  TableBackground
  ClassicButton
  PveButton
  ProfileButton
  RogueliteButton
  RuleBookButton
  CandleProp
  MugProp
  CoinPouchProp
```

## 怎么移动

选中一个节点，比如 `ClassicButton`。

右边 Inspector 里改：

```text
Position -> X / Y
```

记法：

```text
X 变大：往右
X 变小：往左
Y 变大：往上
Y 变小：往下
```

## 怎么改大小

选中节点后，找：

```text
UITransform -> Content Size
```

改：

```text
Width
Height
```

卡片一般保持宽高比例，不要只拉宽或只拉高。比如卡片可以从 `146 x 200` 改到：

```text
135 x 185
155 x 212
```

## 怎么旋转

选中节点，改：

```text
Rotation
```

自然一点的桌面摆法通常是：

```text
卡片：-15 到 15 度
规则书：-20 到 10 度
杯子/钱袋/蜡烛：-8 到 8 度
骰子：0 度或轻微 2 到 4 度
```

## 怎么摆得不那么整齐

不要让东西落在同一条水平线上。

比如不要这样：

```text
ClassicButton.y = 170
ProfileButton.y = 170
```

可以改成：

```text
ClassicButton.y = 185
ProfileButton.y = 145
```

不要让左右两边完全对称。

比如不要这样：

```text
ClassicButton.x = -210
ProfileButton.x = 210
```

可以改成：

```text
ClassicButton.x = -225
ProfileButton.x = 205
```

桌面道具可以压一点边缘，这样更像随手放的：

```text
CandleProp     放左下角或左上角边缘
MugProp        放下方偏左
CoinPouchProp  放下方偏右
RuleBookButton 放右下角
```

## 当前点击事件

每个可点节点都有 `Button` 组件。

看这里：

```text
Button -> Click Events
```

当前绑定是：

| 节点 | 方法 | 作用 |
|---|---|---|
| ClassicButton | `HomeScene.createClassicRoom` | 经典模式，进入 Lobby |
| PveButton | `HomeScene.createPveRoom` | 人机模式，进入 Lobby |
| RogueliteButton | `HomeScene.createRogueliteRoom` | 播放骰面切换动画，然后进入肉鸽界面 |
| ProfileButton | `HomeScene.openProfile` | 打开玩家档案弹窗 |
| RuleBookButton | `HomeScene.openRuleGuide` | 打开规则书弹窗 |
| CandleProp | `HomeScene.tapCandle` | 只做小反馈 |
| MugProp | `HomeScene.tapMug` | 只做小反馈 |
| CoinPouchProp | `HomeScene.tapCoinPouch` | 只做小反馈 |

## 骰子动画

`Canvas` 上的 `HomeScene` 组件里有：

```text
Dice Roll Frames
```

里面挂了 8 张：

```text
dice_roll_01.png
dice_roll_02.png
...
dice_roll_08.png
```

点击骰子时，脚本会快速切这些图片，形成骰面变化。

如果你觉得太快或太慢，改：

```text
Dice Roll Duration
```

推荐范围：

```text
0.45 到 0.9
```

## 服务器没开时为什么也会跳场景

首页现在是这样的逻辑：

```text
先请求服务器创建房间
如果服务器没及时返回，就先进入对应界面预览
服务器返回后，再由 GameManager 接管正式状态
```

这样你做 UI 的时候不会被网络层卡住。
