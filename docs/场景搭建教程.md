# Cocos Creator 场景搭建教程（中文版）

本教程对应 Cocos Creator 3.8.8 中文界面。操作目标是创建 4 个场景，挂好脚本，配好 Build Settings。

---

## 准备工作

打开 Cocos Creator，确认左下角 **资源管理器** 能看到：

```
assets/scripts/scenes/   ← HomeScene、LobbyScene、BattleScene、RogueliteScene
assets/scripts/core/     ← GameManager、ServerActions
assets/scripts/network/  ← NetworkManager
assets/scripts/ui/       ← BattleSeat、DicePanel、ActionSlots 等
```

---

## 第一步：创建 4 个空场景

重复以下操作 4 次，分别创建 Home、Lobby、Battle、Roguelite 四个场景：

1. 在 **资源管理器** 的 `assets` 上右键 → **新建** → **Scene**
2. 把新出现的场景文件重命名为对应的名字（双击文件名即可改名）：
   - 第一个改成 `Home`
   - 第二个改成 `Lobby`
   - 第三个改成 `Battle`
   - 第四个改成 `Roguelite`

> 改名时不要输入 `.scene` 后缀，编辑器会自动加。

完成后的目录结构：

```
assets/
  Home.scene
  Lobby.scene
  Battle.scene
  Roguelite.scene
  scripts/
    ...
```

---

## 第二步：配置场景里的 Canvas 节点

每当你双击打开一个场景，Cocos 会自动在 **层级管理器**（左上角）里生成一个 Canvas 节点和一个 Camera 子节点。不用动 Camera，只需操作 Canvas。

### 2.1 四个场景都需要做的：调整 Canvas 分辨率

1. 在 **层级管理器** 中选中 `Canvas`
2. 看右侧 **属性检查器**，找到 `Canvas` 组件
3. 把 **Design Resolution** 的宽设为 `1280`，高设为 `720`

> 四个场景都做一遍。

---

## 第三步：逐个配置场景

### 3.1 Home 场景

双击打开 `Home.scene`。

**挂 HomeScene 脚本：**

1. 选中 `Canvas` 节点
2. 右侧 **属性检查器** 底部 → 点击 **添加组件**
3. 选 **自定义脚本** → **HomeScene**

**添加按钮（三种模式各一个）：**

在 **层级管理器** 里，右键 `Canvas` → **创建** → **UI 组件** → **Button**，创建三个按钮。选中每个按钮，在属性检查器顶部把名字改成：

| 按钮原名 | 改成 |
|---|---|
| Button | `ClassicButton` |
| Button-001 | `PveButton` |
| Button-002  | `RogueliteButton` |

然后展开每个按钮节点（点左边三角形），选中子节点 `Label`，在属性检查器里把 `String` 分别改成 `经典混战`、`PVE 单人`、`肉鸽爬塔`。

**绑定按钮事件：**

选中 `Canvas`，在属性检查器里找到 `HomeScene` 组件，你会看到三个方法槽位——但目前没有槽位，按钮事件需要这样绑：

对每个按钮：

1. 选中按钮节点（如 `ClassicButton`）
2. 在属性检查器里找到 `Button` 组件
3. 找到 `ClickEvents` 区域，把数组大小改成 `1`
4. 把 **层级管理器** 里的 `Canvas` 拖到新出现的 `cc.Node` 槽位
5. 在右侧下拉框里选 `HomeScene` → 对应的方法：
   - ClassicButton → `createClassicRoom`
   - PveButton → `createPveRoom`
   - RogueliteButton → `createRogueliteRoom`

> 如果没有出现 HomeScene 的方法，说明第 3 步的脚本没挂好，重新做一遍。

**保存：** `Ctrl + S`

---

### 3.2 Lobby 场景

双击打开 `Lobby.scene`。

**挂 LobbyScene 脚本：**

选中 `Canvas` → **添加组件** → **自定义脚本** → **LobbyScene**。

**添加状态 Label：**

右键 `Canvas` → **创建** → **UI 组件** → **Label**，改名为 `StatusLabel`。

**连线：**

选中 `Canvas`，在属性检查器找到 `LobbyScene` 组件，把 `StatusLabel` 节点拖进 `Status Label` 槽位。

**保存：** `Ctrl + S`

---

### 3.3 Battle 场景

双击打开 `Battle.scene`。

#### 3.3.1 挂 BattleScene 脚本

选中 `Canvas` → **添加组件** → **自定义脚本** → **BattleScene**。

添加一个 Label 改名为 `PhaseLabel`，连线到 BattleScene 的 `Phase Label` 槽位。

#### 3.3.2 添加 BattleSeat（两个，一左一右）

右键 `Canvas` → **创建** → **空节点**，改名为 `Seat0`。  
选中 `Seat0` → **添加组件** → **自定义脚本** → **BattleSeat**。  
在属性检查器里把 `Player Index` 设为 `0`。

再右键 `Canvas` → **创建** → **空节点**，改名为 `Seat1`。  
选中 `Seat1` → **添加组件** → **自定义脚本** → **BattleSeat**。  
把 `Player Index` 设为 `1`。

然后给每个 Seat 添加子 Label（右键该 Seat 节点 → **创建** → **UI 组件** → **Label**）：

| 给谁加 | 子节点改名 | String 默认值 |
|---|---|---|
| Seat0 | `NameLabel` | 空位 |
| Seat0 | `HpLabel` | -- / -- |
| Seat0 | `ShieldLabel` | (空) |
| Seat0 | `CharLabel` | (空) |
| Seat1 | `NameLabel` | 空位 |
| Seat1 | `HpLabel` | -- / -- |
| Seat1 | `ShieldLabel` | (空) |
| Seat1 | `CharLabel` | (空) |

连每个 Seat 组件的槽位：选中 `Seat0`，在 BattleSeat 组件里把对应子节点拖进去，Seat1 同理。

#### 3.3.3 添加 DicePanel

右键 `Canvas` → **创建** → **空节点**，改名为 `DicePanelNode`。  
选中此节点 → **添加组件** → **自定义脚本** → **DicePanel**。

在这个节点下创建两个 Label 和一个 Button：

| 类型 | 改名 | 用途 |
|---|---|---|
| Label | `DiceLabel` | 显示骰子结果 |
| Label | `DiceStatusLabel` | 显示"等待投骰" |
| Button | `RollButton` | 点击掷骰 |

展开 `RollButton`，选子节点 Label，把 String 改成 `掷骰`。

选中 `DicePanelNode`，在 DicePanel 组件里把对应的节点拖进 `Dice Label`、`Status Label`、`Roll Button` 槽位。

#### 3.3.4 添加 ActionSlots

右键 `Canvas` → **创建** → **空节点**，改名为 `ActionSlotsNode`。  
选中此节点 → **添加组件** → **自定义脚本** → **ActionSlots**。

在这个节点下创建三个 Button 和一个 Label：

| 类型 | 改名 | 子节点 Label String |
|---|---|---|
| Button | `AttackButton` | 普通攻击 |
| Button | `SkillButton` | 角色技能 |
| Button | `SummonerButton` | 召唤师技能 |
| Label | `ActionHintLabel` | (空) |

选中 `ActionSlotsNode`，在 ActionSlots 组件里：

- 拖 `AttackButton` → `Attack Button`
- 拖 `SkillButton` → `Character Skill Button`
- 拖 `SummonerButton` → `Summoner Skill Button`
- 拖 `ActionHintLabel` → `Hint Label`

#### 3.3.5 添加 BattleLog

右键 `Canvas` → **创建** → **空节点**，改名为 `BattleLogNode`。  
选中此节点 → **添加组件** → **自定义脚本** → **BattleLog**。

在此节点下创建 Label，改名为 `LogLabel`，把 String 清空。

选中 `BattleLogNode`，把 `LogLabel` 拖进 `Log Label` 槽位。

**保存：** `Ctrl + S`

---

### 3.4 Roguelite 场景

双击打开 `Roguelite.scene`。

**挂 RogueliteScene 脚本：**

选中 `Canvas` → **添加组件** → **自定义脚本** → **RogueliteScene**。  
添加 Label 改名为 `PhaseLabel`，连线。

**添加 RoguelitePanel：**

右键 `Canvas` → **创建** → **空节点**，改名为 `RoguelitePanelNode`。  
选中此节点 → **添加组件** → **自定义脚本** → **RoguelitePanel**。

在此节点下创建三个 Label：

| 改名 | 默认 String |
|---|---|
| `StageLabel` | 未开始 |
| `GoldLabel` | 金币 0 |
| `BuffsLabel` | (空) |

选中 `RoguelitePanelNode`，把三个 Label 分别拖进对应的槽位。

**保存：** `Ctrl + S`

---

## 第四步：设置 Build Settings（场景列表）

1. 顶部菜单栏 → **项目** → **构建发布**
2. 在打开的面板里找到 **参与构建场景** 区域
3. 如果列表是空的或不对，点 **刷新** 按钮让它扫描
4. 确保 4 个场景都在列表里，且 **Home 在最上面**（第一个场景就是启动场景）
5. 场景名必须严格等于脚本里的名字：

| 场景文件 | 场景名（在列表里显示的名字） |
|---|---|
| Home.scene | Home |
| Lobby.scene | Lobby |
| Battle.scene | Battle |
| Roguelite.scene | Roguelite |

> 如果名字不对，在 **资源管理器** 里单击选中场景文件，右侧属性检查器里可以看到并修改场景名。

6. 不需要真的构建，关掉面板即可。这个列表只是告诉 Cocos 哪些场景可以被 `director.loadScene` 加载。

---

## 第五步：设置启动场景

1. 双击打开 `Home.scene`
2. 顶部菜单 → **场景** → **设为当前项目启动场景**
3. 保存 `Ctrl + S`

---

## 验证

1. 确认 4 个场景都存了（`Ctrl + S`）
2. 打开 `Home.scene`，点 ▶️ 播放按钮
3. 如果 Cocos 能正常进入 Home 场景、不报错，说明场景配置正确
4. 后面连上服务器后，点击按钮就能创建房间、自动切到 Lobby/Battle 场景

## 常见问题

**Q: 属性检查器里看不到我脚本的槽位？**
先确认 Canvas 节点上确实挂了那个脚本（看属性检查器底部有没有该组件）。如果挂了但看不到属性，检查脚本代码里的 `@property` 装饰器是否正确。

**Q: 点击按钮没反应？**
检查 Button 组件的 `ClickEvents` 是否配置了 `cc.Node`（拖了 Canvas）和正确的方法。

**Q: director.loadScene 报错说找不到场景？**
去 Build Settings（第四步）检查场景名是否完全一致，包括大小写。

**Q: 运行时场景切了但 GameManager 的状态丢了？**
确认 GameManager 所在的节点没有被销毁——它应该在 `onLoad` 里调用了 `director.addPersistRootNode`。
