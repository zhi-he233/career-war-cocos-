# 搭建 05 - 用自己的图片替换灰盒 UI

这一篇接在 `搭建01-04` 后面。

现在 Cocos 里的脚本已经不是空壳了：网络、房间状态、场景跳转、创建房间、加入房间、选角色、开始游戏、掷骰、选择行动、肉鸽奖励/事件/商店/休息/路线都已经接到了脚本层。

接下来你要做的不是重写逻辑，而是把现在脚本自动生成的灰盒文字按钮，慢慢替换成你自己的图片、按钮、头像、血条、面板。

## 先记住一句话

不要先删脚本。

正确顺序是：

1. 先让灰盒 UI 能跑通。
2. 再新建自己的图片节点。
3. 把自己的 Label/Button/Node 拖到脚本组件的属性槽里。
4. 确认能跑，再一点点删掉不需要的灰盒节点。

脚本里很多地方用了 `ensureMinimalUi()`。

它的意思是：如果你没有拖 UI 槽位，脚本会自动生成临时 Label/Button；如果你拖了自己的 UI 节点，脚本就优先用你拖进去的节点。

所以你可以放心自己搭 UI，不用一上来就改代码。

## 图片资源放哪里

建议新建这些文件夹：

```text
assets/
  art/
    ui/
      home/
      lobby/
      battle/
      roguelite/
    characters/
    dice/
    icons/
```

把 PNG 图片拖进对应文件夹。

常见规则：

- 背景图放 `assets/art/ui/...`
- 角色头像放 `assets/art/characters`
- 骰子图放 `assets/art/dice`
- HP、护盾、金币、技能、小图标放 `assets/art/icons`

导入后，选中图片，在 Inspector 里确认它可以作为 `SpriteFrame` 使用。

## 原游戏流程对应 Cocos 场景

原来的游戏大概是这个流程：

```text
HomePage
  -> 创建/加入房间
LobbyPage
  -> 选角色、选召唤师技能、开始游戏
BattlePage
  -> 战斗棋盘、目标、骰子、行动按钮、日志
Roguelite 相关面板
  -> 地图、奖励三选一、事件、商店、休息、继续路线
```

现在 Cocos 里对应：

```text
Home.scene       + HomeScene.ts
Lobby.scene      + LobbyScene.ts
Battle.scene     + BattleScene.ts
Roguelite.scene  + RogueliteScene.ts
```

服务器推什么阶段，`GameManager` 就切到对应场景：

```text
room.phase = lobby              -> Lobby.scene
room.phase = battle/gameOver    -> Battle.scene
room.phase = reward/shop/rest...-> Roguelite.scene
```

你做 UI 时不要自己判断该跳哪个场景，交给 `GameManager`。

## Home 场景怎么换 UI

脚本：`assets/scripts/scenes/HomeScene.ts`

Home 需要这些按钮：

| UI | 作用 | 绑定方法 |
|---|---|---|
| 经典混战按钮 | 创建 classic 房间 | `HomeScene.createClassicRoom` |
| PVE 单人按钮 | 创建 pve_1v1 房间 | `HomeScene.createPveRoom` |
| 肉鸽按钮 | 创建 pve_roguelite 房间 | `HomeScene.createRogueliteRoom` |
| Join Room | 输入房间号后加入 | 脚本可自动生成，也可以拖槽 |
| Refresh Rooms | 刷新房间列表 | 脚本可自动生成，也可以拖槽 |

你有首页图片后，可以这样做：

1. 打开 `Home.scene`
2. 在 `Canvas` 下放一张背景图，比如 `HomeBg`
3. 新建三个按钮图片节点：
   - `ClassicButton`
   - `PveButton`
   - `RogueliteButton`
4. 每个节点加 `Button` 组件
5. Button 的 `ClickEvents` 里拖入 `Canvas`
6. 分别选择上表的方法

如果你想自定义房间号输入框和房间列表：

选中 `Canvas`，看 `HomeScene` 组件，有这些槽位可以拖：

| 槽位 | 拖什么 |
|---|---|
| Status Label | 显示服务器状态的 Label |
| Room Id Edit Box | 输入房间号的 EditBox |
| Join Room Button | 加入按钮 |
| Refresh Rooms Button | 刷新房间按钮 |
| Room List Node | 房间列表容器 Node |

如果不拖，脚本会自动生成临时 UI。

## Lobby 场景怎么换 UI

脚本：`assets/scripts/scenes/LobbyScene.ts`

Lobby 负责：

- 显示房间状态
- 显示玩家列表
- 选择角色
- 选择召唤师技能
- 开始游戏

`LobbyScene` 可拖槽位：

| 槽位 | 推荐 UI |
|---|---|
| Status Label | 顶部房间号、模式、人数 |
| Player List Label | 玩家列表文本，先用 Label 顶着 |
| Selection Label | 当前选择提示 |
| Character List Node | 角色按钮容器 |
| Skill List Node | 召唤师技能按钮容器 |
| Start Game Button | 开始游戏按钮 |

现在最稳的做法：

1. 先自己做 Lobby 背景、标题、玩家列表框。
2. 保留 `Character List Node` 和 `Skill List Node` 两个空容器。
3. 让脚本继续在容器里面生成角色按钮和技能按钮。
4. 等流程稳定后，再把角色按钮改成 Prefab。

以后如果你有角色卡图片，可以做：

```text
CharacterCard.prefab
  Background(Sprite)
  Avatar(Sprite)
  NameLabel(Label)
  SelectedFrame(Sprite)
```

然后再改 `LobbyScene.renderCharacterButtons()`，让它生成角色卡 Prefab，而不是纯文字 Button。

## Battle 场景怎么换 UI

脚本：`assets/scripts/scenes/BattleScene.ts`

Battle 是最大块，建议分成几个 UI 模块做。

### 1. 战斗座位 BattleSeat

脚本：`assets/scripts/ui/battle/BattleSeat.ts`

建议你做一个 `BattleSeat.prefab`：

```text
BattleSeat
  Avatar(Sprite)
  NameLabel(Label)
  CharacterLabel(Label)
  StatusLabel(Label)
  HpBar(ProgressBar)
  HpLabel(Label)
  ShieldLabel(Label)
  RollLabel(Label)
  DeadMask(Node/Sprite)
```

挂 `BattleSeat` 脚本后，拖这些槽：

| 槽位 | 拖什么 |
|---|---|
| Player Index | 第几个玩家，0 是第一个，1 是第二个 |
| Name Label | 玩家名 |
| HP Label | 血量文字 |
| Shield Label | 护盾文字 |
| Character Label | 角色/技能文字 |
| Status Label | 当前状态 |
| Roll Label | 最近骰点 |
| HP Bar | 血条 ProgressBar |
| Dead Mask | 死亡遮罩 |

如果是 1v1，放两个 BattleSeat：

```text
PlayerSeat  -> Player Index = 0
EnemySeat   -> Player Index = 1
```

### 2. 骰子 DicePanel

脚本：`assets/scripts/ui/battle/DicePanel.ts`

建议做：

```text
DicePanel
  DiceImage(Sprite)    可先不用脚本控制
  DiceLabel(Label)     先显示数字
  StatusLabel(Label)
```

拖槽：

| 槽位 | 拖什么 |
|---|---|
| Dice Label | 骰点数字 |
| Status Label | Ready / Choose action |

现在骰子动画是数字跳动。以后如果你有 1-6 的骰子图片，可以在 `DicePanel` 里加 `Sprite` 槽位，把数字换成图片。

### 3. 行动按钮 ActionSlots

脚本：`assets/scripts/ui/battle/ActionSlots.ts`

建议做：

```text
ActionSlots
  HintLabel(Label)
  AttackButton(Button)
  CharacterSkillButton(Button)
  SummonerSkillButton(Button)
```

拖槽：

| 槽位 | 作用 |
|---|---|
| Attack Button | 普通攻击 |
| Character Skill Button | 角色技能 |
| Summoner Skill Button | 召唤师技能 |
| Hint Label | 当前提示 |

服务器会告诉客户端当前有哪些 `availableActions`。

所以按钮不要自己决定能不能点，脚本会根据服务器状态控制显示和禁用。

### 4. 战斗日志 BattleLog

脚本：`assets/scripts/ui/battle/BattleLog.ts`

建议做：

```text
BattleLogPanel
  LogLabel(Label)
```

拖槽：

| 槽位 | 作用 |
|---|---|
| Log Label | 显示最新战斗日志 |
| Max Lines | 最多显示几行 |

现在日志已经按最新时间排序，不会一直显示最旧日志。

## Roguelite 场景怎么换 UI

脚本：`assets/scripts/scenes/RogueliteScene.ts`

这个场景负责非战斗肉鸽阶段：

| room.phase | UI |
|---|---|
| reward | 奖励三选一 |
| roguelite_event | 事件选择 |
| roguelite_shop | 商店购买 |
| roguelite_rest | 休息动作 |
| roguelite_continue | 选择下一关路线 |

现在 `RogueliteScene` 会自动生成文字按钮，先能跑。

如果你要换图，先做这些：

```text
Roguelite.scene
  Background(Sprite)
  TopStatusPanel
    PhaseLabel(Label)
    SummaryLabel(Label)
  ActionList
  LogPanel
    LogLabel(Label)
```

拖到 `RogueliteScene`：

| 槽位 | 拖什么 |
|---|---|
| Phase Label | 阶段/关卡标题 |
| Summary Label | 金币、最近奖励 |
| Action List Node | 奖励/事件/商店/路线按钮容器 |
| Log Label | 日志 |

以后可以把 `ActionList` 里的文字按钮换成：

- `RewardCard.prefab`
- `EventChoice.prefab`
- `ShopItem.prefab`
- `MapNode.prefab`

但第一步不用急，先让流程跑完。

## 哪些脚本是“状态中心”

你做 UI 时，最好知道这几个文件在干什么：

| 文件 | 作用 |
|---|---|
| `core/GameManager.ts` | 全局房间状态、自动切场景、广播状态更新 |
| `network/NetworkManager.ts` | Socket.IO 连接、emit、on |
| `core/ServerActions.ts` | 所有服务器事件封装 |
| `shared/` | 从原项目迁过来的共享战斗/肉鸽逻辑和类型 |
| `helpers/` | 从原 Vue 组件拆出来的显示辅助函数 |
| `models/ViewModels.ts` | 给 UI 用的数据结构 |

简单理解：

```text
服务器 roomUpdate
  -> GameManager 保存 room
  -> 场景/组件收到更新
  -> UI 根据 room 重新显示
```

UI 不应该自己结算伤害。

客户端只负责显示、选择、发请求。真正结算仍然在服务器。

## 你自己做 UI 时最推荐的顺序

### 第一轮：只换背景和大按钮

先换：

- Home 背景
- Home 三个模式按钮
- Lobby 背景
- Battle 背景

不要动战斗逻辑。

### 第二轮：换战斗主界面

再做：

- `BattleSeat.prefab`
- `DicePanel`
- `ActionSlots`
- `BattleLog`

先让它显示文字，图片后补。

### 第三轮：换 Lobby 角色选择

再做：

- 角色卡
- 技能卡
- 玩家列表
- 开始按钮

### 第四轮：换肉鸽

最后做：

- 肉鸽地图
- 奖励三选一
- 商店卡片
- 事件选择
- 休息点

肉鸽 UI 信息最多，最后做会轻松很多。

## 不建议现在做的事

暂时不要：

- 不要把 `GameManager`、`NetworkManager` 从场景里删掉。
- 不要让客户端自己扣血。
- 不要把服务器事件名散落在 UI 按钮里，统一走 `ServerActions`。
- 不要一口气把所有灰盒节点删干净。
- 不要先追求最终美术，先保证每个按钮能跑通。

## 这一版脚本检查结果

我检查并修了几处脚本问题：

- `RogueliteHelpers.ts` 的奖励类型集合太窄，导致 TypeScript 编译报错，已修。
- `ViewModels.ts` 缺少 `enemyTraits` 字段，已补。
- `BattlePlayerHelpers.lastRollText()` 原来会取旧骰点，已改成按最新日志取。
- `NetworkManager` 断线/连接失败后现在会释放连接状态，方便重新连接。
- `DicePanel` 单独挂节点时会自动生成基础 Label，不用必须手拖。

当前 `npx tsc --noEmit --pretty false` 已通过。

## 下一步你可以怎么做

你现在最适合先做 Home 和 Battle 的静态 UI。

建议从这个开始：

1. 在 `assets/art/ui/home` 放首页背景和三个按钮图。
2. 打开 `Home.scene`，把背景和按钮摆好。
3. 给三个按钮绑定 `HomeScene.createClassicRoom`、`HomeScene.createPveRoom`、`HomeScene.createRogueliteRoom`。
4. 跑一次预览，确认能进 Lobby。
5. 再打开 `Battle.scene`，先做两个座位框、一个骰子框、三个行动按钮的位置。

做到这里，你的项目就从“脚本能跑”变成“能开始长成自己的游戏界面”了。
